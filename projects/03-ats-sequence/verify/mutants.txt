# Mutants for 03. Run: ./tools/mutate.sh projects/03-ats-sequence
#
# Format:  <target file> | <sed expression> | <description>
#
# One mutant per real behaviour the code claims to have. Every one MUST make the suite go red. A
# survivor means that behaviour is either dead code or untested, and both are findings.
#
# 03 IS A THIRD SHAPE, AND THE MUTANTS SHOW IT. 01's engine answered a structural question, so its
# mutants broke graph logic. 02's whole claim was arithmetic, so its mutants were one character edits
# to an equation. This engine's claim is that a STATE MACHINE reaches published states at published
# times, so these mutants come in two flavours: loosen a published boundary, or quietly fill in a
# number the manufacturer does not publish.
#
# THE SECOND FLAVOUR IS THE POINT OF THIS PROJECT. Four of these (extended parallel value, parallel
# duration, in phase angle, shed load contradiction) do not break a calculation at all. They make the
# tool CLAIM MORE THAN ITS SOURCE DOES, which is the exact failure this repo exists to avoid and the
# one that no ordinary test suite would ever look for. If any of those four survives, the refusal it
# guards is decoration and someone will delete it.
#
# ⚠ RUNNER CONSTRAINT, learned here: mutants.txt is parsed with `IFS='|' read`, so A SED EXPRESSION
# MAY NOT CONTAIN A PIPE. `s/a && b/a || b/` silently truncates at the first `||` and the mutant runs
# as garbage. Express "any one of these suffices" by DROPPING each criterion in turn instead, which
# is a better test anyway: it proves each criterion is individually load bearing.
#
# ⚠ DELIBERATELY NOT MUTATED, and the reason is worth reading: the 1C comparison
# `s.normalReturnsAt < tOverride`. Loosening it to `<=` changes only what happens at an outage of
# EXACTLY 1.000 s, and ASCO does not publish what the switch does at that instant. The sync
# criterion's boundary IS published ("must be LESS THAN 5 degrees"), so it is asserted at the
# boundary and mutated there. This one is not. Adding the mutant would force an assertion about
# unsourced behaviour to kill it, which is how a suite starts inventing facts to keep itself green.
# One boundary is in the book. The other is not. Only mutate the one that is.
